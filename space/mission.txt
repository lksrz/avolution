You have no memory between restarts beyond what you write to disk.
Everything outside `/workspace` is forbidden.

Read `.loop_meta.json` at the start of every loop:
  - `loop_start_utc`  — when this loop started
  - `timeout_sec`     — total budget (300s)
  - `was_killed`      — if True: you were SIGKILL'd last loop, do NOT rewrite from scratch

YOUR MISSION

Build a program that predicts whether the next BTC/USDT 5-minute candle
will close HIGHER (UP) or LOWER (DOWN) than it opened.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PREDICTION RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Time is divided into 5-minute windows aligned to the clock:
  :00, :05, :10, :15, :20, :25, :30, :35, :40, :45, :50, :55

Each window has a DEADLINE = window_start + 4m30s (30s before it closes).

  Example:
    Window 09:00–09:05  →  deadline 09:04:30
    You predict: will the candle 09:05–09:10 be UP or DOWN?
    Verified at: 09:10:00 (when 09:05 candle closes)

HOW TO CALCULATE YOUR WINDOW:
  import time
  now = time.time()
  window_start = (now // 300) * 300
  deadline = window_start + 270          # 4m30s into window
  target_open = window_start + 300       # the candle you're predicting
  target_close = window_start + 600      # verified at this time

STRATEGY — MAKE PREDICTION EARLY, UPDATE IF NEEDED:
  1. At the START of each window (loop after :00), make a prediction immediately.
     A weak early prediction is better than a missed deadline.
  2. In subsequent loops of the same window, you MAY update the prediction
     if new data changes your view. The LAST prediction before deadline counts.
  3. Never wait until the deadline — SIGKILL may come before you get there.

VERIFICATION:
  At the start of each loop, check if the target candle has closed.
  Use Binance API: GET /api/v3/klines?symbol=BTCUSDT&interval=5m&limit=3
  Compare: close > open → UP, else DOWN.
  Match against your last committed prediction for that candle.

PREDICTIONS LOG FORMAT:
  Append one line per window to `predictions.log`:
  <ISO-timestamp> | window=09:00 | target=09:05 | prediction=UP | actual=DOWN | correct=False

INTEGRITY:
  - Never modify or delete previous lines in predictions.log
  - Prediction is committed to git before the target candle opens → tamper-proof record
